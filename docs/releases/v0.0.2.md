# v0.0.2

## Summary

This release adds per-question timeout warnings, dynamic deadline footers on question text, kanji-perfect answer bonuses, and an outro date header. It also propagates outbound message keys through the sender chain so delayed system messages can reply to earlier bot messages.

## What was implemented

### Normal question timeout

- Increased normal-stage question timeout from 60 to **90 minutes**.
- Special (God) stage timeout remains 30 minutes.

### Per-question warning at T-10 minutes

- A warning is scheduled when each question is posted (including God stage).
- Fires 10 minutes before the question deadline with exact text:

```
⏰ Tinggal 10 menit lagi!
```

- Warning message replies to (quotes) the original question message.
- Warning timer is cancelled when the question is answered, times out, or the quiz stops.

### Dynamic deadline footer on question text

- Each question now ends with a footer showing the WIB deadline time:

```
⏰ HH.MM.SS WIB
```

- Time is computed from the real per-question deadline (accounts for delays before posting).

### Kanji-perfect answer bonus

- Correct answers containing kanji characters (`\p{Script=Han}`) now trigger:
  - Reaction: 🌸 (instead of ✅)
  - Winner header: `🤩 かんぺきだった！` (instead of `🤗 せいかいだった！`)
  - Additional **+2 points** on top of existing correct-answer scoring.
- Example: normal cap question answered with kanji = `12 pts` (10 cap + 2 kanji bonus).

### Outro date header

- Final scoreboard now includes the same JP weekday + Indonesian date format as intro:

```
🗓️ 土︱27 September 2025
```

- Placed between the `🏁 END` header and the points list.

### Outbound message key propagation

- Sender interfaces now return the message key of sent messages (`OutgoingMessageKey`).
- This enables the engine to quote outbound messages in later system messages (e.g., warning reply to question).
- Updated across:
  - `IWhatsAppClient.sendText` / `sendImageWithCaption`
  - `WhatsAppClient` wrapper
  - `WWebJsWhatsAppClient` (extracts `_serialized` from wwebjs response)
  - `BaileysWhatsAppClient` (extracts `key` from Baileys response)
  - `DaemonRuntime.enqueueOutbound` (propagates key through queue)
  - `QuizEngine.SenderPort`
