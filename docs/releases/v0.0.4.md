# v0.0.4

## Summary

This release adds the **season** feature for accumulating quiz points across multiple quizzes within a group, renames `kotaete run` to `kotaete quiz run`, and includes recent fixes for `--save-svg` and scheduling guardrails.

## What was implemented

### Season feature

- New `season` config property in `kotaete.ts` with shape:
  ```ts
  season?: {
    start?: boolean    // Reset accumulated season points for group before this quiz
    end?: boolean      // Send special congrats messages + scoreboard image after final scoreboard
    caption?: string   // Caption for season-end top-3 message
    scoreboardTemplate?: string  // Path to SVG template (default: ~/.kotaete/scoreboard-template.svg)
  }
  ```
- Points accumulate across quizzes per group (persisted via `~/.kotaete/state/season-points.json`; survives daemon restarts).
- `season.start === true` resets accumulated season points before applying this quiz's points.
- Normal end-of-quiz scoreboard remains per-quiz only (existing behavior unchanged).
- When `season.end === true`, immediately after the normal final scoreboard:
  - Sends top-3 congratulation message with medals (🥇🥈🥉).
  - Generates and sends a scoreboard image (via SVG template + Inkscape PNG render, then `magick` JPG quality 85) showing top 7 ranked members; excess slots use `-` placeholders.
  - Replaces `{{caption}}` placeholder in scoreboard template using `season.caption` when provided.
  - Sends "others" message with remaining participants and their season points.
  - Retains season accumulated points (they are reset at the start of the next season).

### Command path rename

- `kotaete run` → `kotaete quiz run` (now a subcommand of `kotaete quiz`).
- Top-level `run` registration removed from root CLI.
- Flags and behavior remain unchanged.

### Season scoreboard image generation

- New helper module `src/quiz/season-scoreboard.ts` with:
  - SVG template rendering (replaces `{{rankN}}`, `{{kanaN}}`, `{{romajiN}}`, `{{classgroupN}}`, `{{scoreN}}`, and `{{caption}}` for slots 1–7).
  - Inkscape PNG export + ImageMagick JPG conversion (`-quality 85`).
  - Default template path: `~/.kotaete/scoreboard-template.svg` (overridable via `season.scoreboardTemplate`).
- Output images are written in quiz directory (`season-scoreboard.png` and `season-scoreboard.jpg`); JPG is used for WhatsApp send.

### Recent fixes (carried over)

- **Member identity model migration (LID-first)**:
  - `members` now requires `lid` for each member as the canonical identity.
  - `pn` is now optional metadata only (used for display/fallback matching when needed).
  - Quiz runtime state (scores, cooldowns, wrong-streak tracking, special-stage attempts) is keyed by `lid`.
  - Season accumulation is keyed by `lid` and season CLI now displays `lid` as the primary identifier.

- **Baileys inbound parsing regression fix**: Messages wrapped in `ephemeralMessage`, `viewOnceMessage`, `viewOnceMessageV2`, or `editedMessage` are now properly unwrapped to extract text content. Sender JID extraction falls back to `participantAlt` / `remoteJidAlt` when `key.participant` is missing, preventing dropped reactions/replies.
- **`kotaete season` CLI**: New top-level command with subcommands `show`, `add`, `set`, `reset`, `clear` for viewing and manipulating season scores from the terminal. Respects `--json` output.
- `--save-svg` flag keeps rendered SVG in quiz directory alongside generated image for debugging.
- **`--no-generation` / `--generation` on `kotaete quiz run` only**: image generation toggle is scoped to quiz run path. Default is no generation; use `--generation` to enable SVG rendering.
- Question and season images follow two-step export: Inkscape SVG→PNG then `magick` PNG→JPG (quality 85). JPG is used for WhatsApp delivery while PNG is kept in quiz directory.
- Past-start scheduling rejection with clear error message.
- Scheduling guardrail: if intro is past but first round start is future, intro sends immediately and quiz waits for start time.
- **SVG template background fix**: `renderQuestionSvgTemplate()` no longer clobbers all `href`/`xlink:href` attributes in the SVG. Now targets only the question image (first `<image>` element or explicit placeholders like `{{image}}`), preserving background and embedded object references.
- **`--no-schedule` formatting parity**: Markdown-parsed questions now strip leading `Hint:`/`hint:` prefix and append the same `*Opsi jawab:*` options block as scheduled config questions (kana/romaji/kanji availability inferred from answer lines; `+N` extra-points markers supported).
- **Scheduling guard message in English**: The "first round start time has passed" error message is now in English (was Indonesian).
- **File logging with rotation**: Daemon instances write logs to `~/.kotaete/logs/daemon-<pid>.log` with 16 KiB rotation and 5-file retention. Non-daemon commands remain console-only.
- **Baileys inbound debug logging**: Added explicit debug logs before each message drop condition (missing message, fromMe, duplicate, old replay, non-group, empty text/sender) for easier diagnosis of dropped answers.
- **No-active-job drop hint**: Daemon now appends a fresh-mode hint in debug logs when inbound messages are dropped because no quiz job is active, making `daemon --fresh` operational order clearer.
- **Answer ending validation fix**: Quiz inbound answer filter now accepts Unicode letters/numbers/combining-marks and Katakana long vowel mark (`ー`) as valid final characters, preventing valid Japanese answers (e.g. `スーパー`) from being dropped before reaction/reply handling.
