# v0.0.1

## Summary

This release establishes the production-ready quiz daemon/relay foundation and ships the first full set of runtime hardening updates: provider abstraction, LID→PN identity handling, reconnect safety, outbound pacing, improved quiz controls, and v0.0.1 scoring/stage behavior updates.

## What was implemented

### Daemon + Relay architecture

- Single CLI surface for:
  - `kotaete daemon`
  - `kotaete run <groupId> <quizDir> <membersFile>`
  - `kotaete quiz status`
  - `kotaete quiz stop`
- Daemon request validation through zod schemas.
- Runtime metadata tracking for active quiz status reporting.

### WhatsApp provider abstraction

- Added provider abstraction with:
  - `wwebjs` as default provider
  - `baileys` as experimental provider
- Added provider-aware startup/auth defaults.
- Added support for quoted/reply sending in text responses.
- Added typing API in client interface (`sendTyping(groupId)`).

### Persistent LID ↔ PN mapping

- Implemented persistent JSON-backed map under state dir (`~/.kotaete/state/lid-pn-map.json`).
- PN-first sender resolution flow for inbound messages.
- Mapping reuse across both providers.

### Reconnect and inbound safety

- Generation-based stale callback protection.
- Replay protection via message-id dedupe and timestamp cutoff.
- Additional debug logs for inbound drops, resolution source, and lifecycle transitions.

### Quiz flow and UX updates

- Answer-ending filter: ignore messages that do not end with Latin/Japanese letters.
- Cooldown warning now sent once per question with reply quoting.
- Winner message now replies to the winning answer.
- Cooldown toggle UX on `run`:
  - default cooldown on
  - supports `--no-cooldown`

### Outbound queue behavior

- Text/image outbound actions are serialized with 5-second pacing.
- Typing indicator is sent before text/image sends.
- Typing now starts right before outbound wait finalization and keeps at least 2 seconds before send.
- Reactions are no longer throttled by outbound queue pacing.

### Quiz content loading

- Quiz loader now supports note files from quiz directory:
  - `intro.md` as intro footer note (preferred)
  - existing start-note fallback names remain supported
  - `outro.md` as final footer content

### Intro/outro formatting

- Intro day line format aligned to:
  - `土︱27 September 2025`
- Final scoreboard footer now uses `outro.md` content when present, with existing default fallback when absent.

### Scoring updates (v0.0.1 rules)

- Normal stage:
  - wrong answer gives `+1`
  - correct answer gives only the remaining points up to max `10` total for that question
  - example: 2 wrong + 1 correct = `10` total (not `12`)
- Special (God) stage:
  - correct answer gives `+25`
  - wrong answer gives `+0`
  - one attempt per member

### Special (God) stage behavior

- Special stage no longer shows normal `(x/y)` progress counter.
- Added pre-stage announcement message:

```md
🪽 _神のステージ! (Stage Dewa!)_
Khusus stage ini, siapa pun bisa menjawab soal. Ketentuannya yaitu:
🌸 Jawaban benar = +25 poin
🙊 Satu kesempatan per anggota
⏰ Masa aktif soal 30 menit
Akan dimulai dalam 1 menit!
```

- After announcement, bot waits 1 minute before sending special-stage question.
- Stage time limits:
  - normal stage: 1 hour
  - special stage: 30 minutes
